方法与边界

系统架构

Sensing → StateObject → NurseAgent(规则) → DiagnosisAgent(LLM) → TriageDecision → EpisodeLog

各模态职责

WiFi CSI-based 呼吸/心率估计，受 NLOS 影响较小

mmWave 毫米波雷达，跌倒检测 + 生命体征，NLOS 下降级

Thermal 红外体温

Agent 分工

NurseAgent — 规则引擎。8 条硬规则：跌倒、HR/RR/体温异常（z-score 优先）、低置信度、NLOS、模态冲突、RR 基线偏离。

FusionEngine — 三步链：每模态置信度 → 跨模态一致性 → 仲裁输出。portable_v2 数据存在时使用真实每模态估计值。

DiagnosisAgent — LLM 分级。反射弧(path)处理 L4 紧急事件，ReAct 循环处理其余事件。

引用来源

RCP_NEWS2_2017_REFERENCE HR/RR/Temperature 绝对偏离参考区间。注意：当前系统缺乏 SpO₂、收缩压和意识评估参数，不计算完整 NEWS2 评分。

NICE_NG249_2025 跌倒评估证据需求。系统通常不知道 injury、意识丧失、能否独自起身、反复跌倒历史。

RESIDENT_HISTORY 个人纵向基线数据，统计偏离证据不等于疾病诊断。

SENSOR_FUSION 测量可靠性证据，不是疾病证据。

PROJECT_POLICY 项目特有的 z-score 阈值、5–10 分钟复查策略、L0-L4 映射。

安全边界

本系统提供 care support，不构成医学诊断、治疗建议或用药建议。L0-L4 分级为项目策略，不应用于独立临床决策。
